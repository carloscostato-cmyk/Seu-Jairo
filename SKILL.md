# Skill de Execução do Agente (Prompt Intake)

Este arquivo define o comportamento padrão de execução para mudanças no projeto.

## Regra Base
1. Ao receber um novo prompt, levantar dúvidas objetivas quando necessário.
2. Antes de começar a codar, confirmar explicitamente: "Posso codar? (sim/não)".
3. Se a resposta for "sim", executar implementação completa com validação e feedback final.
4. No fechamento da execução, sempre informar a URL do dashboard (.io) para validação no navegador.

## Checklist de Entrega
- Sem erros de sintaxe/lint nos arquivos alterados.
- Sem links inválidos evidentes no dashboard e nas vagas persistidas.
- Commit e push realizados quando aprovado.
